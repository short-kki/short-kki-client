@components/ShortsFeed.tsx

현재 작성된 `ShortsFeed` 컴포넌트의 심각한 문제점들을 수정하고, **인스타그램 릴스(Reels)와 똑같은 UX**로 다시 구현해줘.

[현재 문제점]
1. **자동 재생 실패:** 영상이 바로 재생되지 않고 썸네일과 재생 버튼이 뜸.
2. **화면 잘림:** 16:9 영상이 9:16 화면에 들어가면서 위아래 거대한 검은 여백(레터박스)이 생김.
3. **UI 간섭:** 유튜브 기본 UI가 화면을 가리고 터치를 방해함.

[필수 수정 요구사항] (매우 중요)

**1. 자동 재생 보장 (Auto-play Policy)**
- 모바일 OS 정책상 소리가 켜진 상태로는 자동 재생이 차단됨.
- **초기 상태를 반드시 `mute={true}`(음소거)**로 설정해서 강제로 자동 재생되게 할 것.
- 대신, 화면을 터치하면 소리가 켜지도록(`setIsMuted(false)`) 투명 버튼이나 오버레이 UI를 추가해줘.

**2. 화면 꽉 채우기 (Zoom & Cover)**
- `YoutubePlayer`를 감싸는 View에 **`transform: [{ scale: 1.8 }]` (또는 1.7)** 스타일을 적용해줘.
- 이렇게 해서 영상을 강제로 확대해 위아래 레터박스를 화면 밖으로 밀어내고, 중앙 영상만 꽉 차게 보여줘.

**3. 유튜브 UI 숨기기 & 터치 방지**
- `YoutubePlayer` 영역에 `pointerEvents="none"`을 적용해서 사용자가 유튜브 제목이나 일시정지 버튼을 누르지 못하게 막아줘.
- 오직 스크롤(Swipe)만 가능하게 하고, 재생 컨트롤은 우리 앱의 오버레이 UI가 담당하게 해줘.

**4. 스크롤 로직 (Snap & Play)**
- `FlatList`의 `pagingEnabled`를 활성화.
- `onViewableItemsChanged`를 사용하여 **현재 화면에 80% 이상 보이는 인덱스(`playingIndex`)만 `play={true}`**가 되도록 로직 정교화.
